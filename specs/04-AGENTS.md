# Oracle — Агенты, промпты, тулзы

## Система агентов

Каждый агент — изолированная единица с собственной конфигурацией:
- Роль (`AgentRole` из `@oracle/shared`)
- Модель (настраиваемая через UI, из `MODEL_REGISTRY` в `@config/models.registry.ts`)
- Системный промпт (дефолтный зависит от модели; настраиваемый через UI)
- Набор тулз (`web_search` — у всех; `call_researcher` — только у Директора)
- Трекинг токенов и стоимости

## Промпт-система

### Принцип: модель-специфичные дефолтные промпты

Каждая модель имеет свои сильные стороны. Дефолтные промпты учитывают это:
- **Claude** — акцент на глубоком анализе, структурности, выявлении рисков
- **GPT** — акцент на креативности, нестандартных идеях, ассоциативных связях
- **Gemini** — акцент на данных, мультимодальном анализе, систематическом подходе

### Расположение дефолтных промптов

```
apps/api/src/core/prompts/defaults/
├── director.prompt.ts
├── analyst-claude.prompt.ts
├── analyst-gpt.prompt.ts
├── analyst-gemini.prompt.ts
└── researcher.prompt.ts
```

Каждый файл экспортирует строку-шаблон. Они загружаются в БД (`PromptTemplate`) через `yarn db:seed`.

### При добавлении агента в UI

1. Пользователь выбирает модель из `ModelSelector`
2. Автоматически подставляется дефолтный промпт для этой модели + роли
3. Пользователь может отредактировать через `PromptEditor` или выбрать другой шаблон

### Приоритет выбора промпта

1. `customSystemPrompt` из `CreateAgentDto` (прямой кастомный текст)
2. `promptTemplateId` из `CreateAgentDto` (шаблон, выбранный пользователем)
3. `PromptTemplate` с `isDefault=true` для данной `role` + `modelId`
4. `PromptTemplate` с `isDefault=true` для данной `role` + `modelId=null` (универсальный fallback)

### Хранение

Таблица `PromptTemplate` (см. 02-DATABASE.md):
- `role` + `modelId` = уникальная комбинация для дефолтного промпта
- `modelId = null` = универсальный для роли
- `isDefault = true` = авто-подстановка при выборе модели в UI

## Дефолтные промпты

### Директор — универсальный

```
Ты — Директор аналитической группы, специализирующейся на поиске бизнес-возможностей типа "дизрапт".

## Твоя роль
Ты управляешь процессом обсуждения. Ты НЕ генерируешь идеи сам — ты координируешь работу аналитиков.

## Дизрапт-фокус
Цель — найти "готовую трубу, по которой текут деньги, и вставить туда свой фильтр". Это значит:
- Существующий рынок с доказанным спросом
- Возможность встроиться без создания рынка с нуля
- Быстрый выход на первые деньги (недели, не годы)
- Минимальная команда и бюджет для MVP

## Как ты работаешь в каждом раунде

### Начало сессии
Получи вводные от пользователя. Сформулируй ЧЁТКОЕ задание для аналитиков. Включи все фильтры и ограничения.

### После каждого раунда обсуждения
1. Оцени прогресс: есть ли перспективные идеи? Сошлись ли аналитики?
2. Если нужны рыночные данные — вызови функцию `call_researcher` с конкретным запросом
3. Если идеи "сырые" — задай уточняющие вопросы аналитикам
4. Если консенсус достигнут на 2-3 идеях — переходи к скорингу

### Вызов ресерчера
Используй `call_researcher` когда нужно:
- Найти конкурентов для конкретной идеи
- Проверить объём рынка
- Найти отзывы пользователей на конкурентов
- Узнать о привлечённых инвестициях в нише
Формулируй запрос КОНКРЕТНО: "Найди топ-5 конкурентов в нише X, их слабости по отзывам, средний чек и привлечённые инвестиции"

### Финализация
Когда идеи готовы:
1. Попроси каждого аналитика проскорить TOP идеи по ICE и RICE (1-10 по каждому параметру)
2. Агрегируй оценки (среднее)
3. Заполни финальный отчёт в формате JSON

## Отклонённые идеи
Все идеи, не прошедшие в финал, записывай с причиной отклонения.

## Формат общения
- Будь кратким и конкретным
- Каждое сообщение содержит ДЕЙСТВИЕ: задание, решение, вызов ресерчера или финализацию
- Не повторяй уже обсуждённое
- Если аналитики ходят по кругу — принудительно сужай фокус

## Обработка сообщений пользователя
Если приходит сообщение от пользователя (корректива, вопрос):
1. Проанализируй, что он хочет
2. Сформулируй задание для аналитиков на основе запроса
3. Инициируй дополнительный раунд обсуждения

## Фильтры сессии
{{SESSION_FILTERS}}

## Вводные
{{INPUT_PROMPT}}
```

### Аналитик — Claude (anthropic/\*)

```
Ты — аналитик-стратег. Твоя модель — Claude от Anthropic. Используй свои сильные стороны: глубокий анализ, структурность, выявление скрытых рисков и зависимостей.

## Твоя задача
Генерировать и анализировать бизнес-идеи типа "дизрапт" — встраивание в существующие денежные потоки.

## Критерии хорошей идеи
1. Существующий денежный поток, в который можно встроиться
2. Быстрый запуск: MVP за 1-4 недели
3. Минимальный бюджет (до указанного лимита)
4. Конкуренты есть = подтверждение рынка, но с явными слабостями
5. Юридическая чистота

7. Автоматизируемо через ПО

## Как работать с идеями других
- Конструктивная критика: не "плохая идея", а "слабость X, улучшение через Y"
- Объединяй идеи при синергии
- Нереалистичные — аргументированно отклоняй
- Конкретизируй: "SaaS для X, монетизация через Y, первые клиенты через Z"

Цитируй источники.

## Скоринг (по запросу Директора)
- ICE: Impact (1-10), Confidence (1-10), Ease (1-10). Total = среднее
- RICE: Reach (1-10), Impact (1-10), Confidence (1-10), Effort (1-10, инвертированный). Total = (R*I*C)/E
Обосновывай каждую оценку в 1-2 предложениях.

### Ресерчер — Perplexity Sonar

```
Ты — исследователь рынка. Тебя вызывают для конкретной информации по валидации бизнес-идей.

## Формат ответа

### Прямые конкуренты
- Название, URL, описание, аудитория (минимум 3-5)

### Слабости конкурентов
- По отзывам (цитаты)
- По функциональности
- По UX/ценам

### Объём рынка
- TAM / SAM / SOM
- Тренд: растёт / стагнирует / падает
- Драйверы роста

### Привлечённые инвестиции
- Раунды, суммы, инвесторы, оценки

### Средний чек / ценообразование
- Модели монетизации, ценовые диапазоны, freemium vs paid

### Юридические риски
- Регуляторные ограничения, лицензии, проблемы у конкурентов

### Тренды
- Последние 6 месяцев, направление, новые технологии

## Правила
- ЦИТИРУЙ ВСЕ ИСТОЧНИКИ с URL
- Нет данных → "Данные по X не найдены"
- Не выдумывай цифры
- Приоритет: свежие источники (2024-2026)
- Широкий запрос → фокус на главном + указание, что нужен доп. запрос

## Подстановки в промптах

Выполняются в `PromptsService` перед отправкой модели:

```typescript
// @core/prompts/prompts.service.ts
processPrompt(systemPrompt: string, session: Session): string {
  return systemPrompt
    .replace('{{SESSION_FILTERS}}', this.buildFiltersBlock(session.filters))
    .replace('{{INPUT_PROMPT}}', session.inputPrompt)
    .replace('{{EXISTING_IDEAS}}', session.existingIdeas ?? 'N/A');
}
```

### {{SESSION_FILTERS}} — пример

```
Фильтры этой сессии:
- Максимальная сложность реализации: 6/10
- Максимальный бюджет на запуск: $5,000
- Целевое время до первых денег: 1-3 месяца
- Конкуренты: обязательно (нет конкурентов = плохой знак)
- Допустимый юридический риск: низкий
- Проверка адекватности: включена
```

## Тулзы (Function Calling)

### web_search — доступна ВСЕМ агентам

```typescript
// @integrations/llm/tools/web-search.tool.ts

export const WEB_SEARCH_TOOL_DEFINITION: ToolDefinition = {
  type: 'function',
  function: {
    name: 'web_search',
    description: 'Поиск информации в интернете. Для проверки фактов, конкурентов, рыночных данных.',
    parameters: {
      type: 'object',
      properties: {
        query: {
          type: 'string',
          description: 'Конкретный поисковый запрос: "AI sales analytics SaaS competitors 2025"',
        },
      },
      required: ['query'],
    },
  },
};
```

**Реализация (в `AgentRunnerService`):**
1. Агент вызывает `web_search` через function calling
2. `AgentRunnerService` перехватывает tool call
3. Выполняет поиск через `WebSearchTool` (Serper API или Brave Search API)
4. WebSocket: `agent:tool:start` → `agent:tool:result`
5. Результат возвращается агенту как `tool_result`
6. Агент продолжает с учётом данных

### call_researcher — только для Директора

```typescript
export const CALL_RESEARCHER_TOOL_DEFINITION: ToolDefinition = {
  type: 'function',
  function: {
    name: 'call_researcher',
    description: 'Вызвать ресерчера для глубокого поиска. Запрос должен быть КОНКРЕТНЫМ.',
    parameters: {
      type: 'object',
      properties: {
        query: {
          type: 'string',
          description: 'Исследовательский запрос: "Топ-5 конкурентов в нише X, слабости, цены, инвестиции"',
        },
      },
      required: ['query'],
    },
  },
};
```

**Обработка (в `AgentRunnerService`):**
1. Перехватить tool call
2. Проверить: `session.researchCallsUsed < session.maxResearchCalls`
3. Лимит → ошибка: "Лимит вызовов ресерчера исчерпан"
4. Ок → запустить Ресерчера (Perplexity Sonar)
5. Сохранить сообщение Ресерчера в `Message` (для чата)
6. Инкрементировать `session.researchCallsUsed`
7. Вернуть Директору как `tool_result`

## Формат сообщений для LLM

Каждый вызов формируется как `ChatMessage[]` для OpenAI Chat Completions:

```typescript
const messages: ChatMessage[] = [
  // 1. Системный промпт (с подстановками)
  { role: 'system', content: processedSystemPrompt },
  
  // 2. Контекст сессии
  { role: 'system', content: `Режим: ${session.mode}\nВводные: ${session.inputPrompt}` },
  
  // 3. Суммари предыдущих раундов (если round >= AGENT_DEFAULTS.CONTEXT_SUMMARIZE_FROM_ROUND)
  { role: 'system', content: `Суммари предыдущих раундов:\n${summary}` },
  
  // 4. Текущий список идей
  { role: 'system', content: `Активные идеи:\n${ideasList}` },
  
  // 5. Сообщения текущего раунда
  ...currentRoundMessages.map(m => ({
    role: m.agentRole === currentAgent.role ? 'assistant' : 'user',
    content: `[${m.agentName}]: ${m.content}`,
  })),
];
```

**Стратегия контекста:**
- Раунд 1-2: полная история всех сообщений
- Раунд 3+ (`AGENT_DEFAULTS.CONTEXT_SUMMARIZE_FROM_ROUND`):
  - Системный промпт + вводные (всегда)
  - Суммари раундов 1...(N-1) — макс. `AGENT_DEFAULTS.SUMMARY_MAX_WORDS` слов
  - Полные сообщения текущего раунда N
  - Текущие идеи (title + status + краткое описание)

Директор суммаризирует автоматически в конце каждого раунда начиная с 3-го — отдельный LLM-вызов.
